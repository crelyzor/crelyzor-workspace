## Issues Noted (Not Fixed — Require Migration or Architecture Decision)

These were identified by the Schema Reviewer but require design decisions:

1. **15 models missing soft delete fields** — Session, LoginHistory, RefreshToken, OAuthAccount, MeetingParticipant, MeetingStateHistory, TranscriptSegment, MeetingSpeaker, MeetingAIContent, AskAIConversation, AskAIMessage, GCalSyncState, CardView, UserUsage, Subscription. Many are ephemeral/auth records where soft delete may be intentionally omitted.

2. **MeetingNote.author is a dangling UUID string** — should be a proper relation to User. Requires data migration (column rename author -> authorId + FK constraint).

3. **Cascade deletes on core models** — Session, LoginHistory, RefreshToken, OAuthAccount, CardContact, Card use onDelete: Cascade on User relation. Per convention, cascade deletes should only be on junction tables.

4. **taskController.ts bypasses service layer** — Direct Prisma usage in controller with multiple sequential writes without transactions (createTask, createStandaloneTask, updateTask). The GCal sync side-effects are intentionally fail-open, so wrapping everything in a transaction would change semantics. Recommend extracting task service layer in a future refactor.

5. **Redundant indexes** — UserSettings, UserUsage, Subscription, GCalSyncState have @@index on fields that already have @unique (implicit index). Harmless but unnecessary.
